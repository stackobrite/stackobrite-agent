# stackobrite-agent

Lightweight remote execution agent for Stackobrite cloud hoster. Runs on each EC2 instance, binds to `localhost:9090`, and is accessed exclusively via SSM Session Manager port-forwarding. No SSH keys, no public attack surface.

## Architecture

```
Platform Backend â†’ SSM Port-Forward â†’ localhost:9090 â†’ Agent â†’ kubectl/helm/etc
```

The agent never listens on a public interface. All access is tunneled through AWS SSM, authenticated with the instance's IAM role.

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/health` | Health check (returns version) |
| GET | `/version` | Detailed version info (version, build date, git commit, Go version) |
| POST | `/exec` | Execute a whitelisted binary with args |
| POST | `/exec/shell` | Execute a shell command via `bash -c` |
| GET | `/exec/stream` | SSE stream of stdout/stderr from a command |
| GET | `/kubeconfig` | Read the kubeconfig file |
| POST | `/update` | Self-update: download new binary, swap, restart |

All endpoints require `Authorization: Bearer <token>` header.

## Authentication

HMAC token-based auth. The token is:
1. Generated by Terraform as a `random_string` resource
2. Passed to the instance via `user_data.sh`
3. Stored at `/etc/stackobrite/agent.key`
4. Loaded by the agent via `AGENT_TOKEN_FILE` env var

The agent uses constant-time comparison (`crypto/subtle`) to prevent timing attacks.

## Rate Limiting

100 requests per minute per IP. Returns `429 Too Many Requests` with `Retry-After: 1` header when exceeded.

## Request Tracking

Every request gets an `X-Request-ID` header (generated if not provided). All audit log entries include this ID for correlation.

## Command Whitelist

Only these binaries can be executed:

`kubeadm`, `kubectl`, `helm`, `containerd`, `crictl`, `apt-get`, `apt`, `systemctl`, `modprobe`, `sysctl`, `curl`, `wget`, `cat`, `ls`, `cp`, `mkdir`, `chown`, `chmod`, `tee`, `sed`, `gpg`, `unzip`, `tar`, `journalctl`, `dmesg`, `ip`, `ss`, `whoami`, `hostname`, `env`, `printenv`, `rm`, `mv`, `head`, `tail`, `grep`, `bash`, `sh`, `openssl`, `jq`, `base64`, `echo`, `sleep`

## Usage

### Health Check
```bash
curl -H "Authorization: Bearer $TOKEN" http://localhost:9090/health
# {"status":"ok","agent":"stackobrite-agent v0.1.0"}
```

### Execute Command
```bash
curl -X POST -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  http://localhost:9090/exec \
  -d '{"binary":"kubectl","args":["get","pods","-A"]}'
```

### Shell Command
```bash
curl -X POST -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  http://localhost:9090/exec/shell \
  -d '{"command":"kubectl get nodes -o wide"}'
```

### SSE Stream
```bash
curl -N -H "Authorization: Bearer $TOKEN" \
  "http://localhost:9090/exec/stream?binary=kubectl&args=%5B%22logs%22%2C%22-pod-name%22%5D&timeout=30"
```

### Self-Update
```bash
curl -X POST -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  http://localhost:9090/update \
  -d '{"binary_url":"https://sb-tfstate-690976584343.s3.amazonaws.com/agent/stackobrite-agent"}'
```

### Version Check
```bash
curl -H "Authorization: Bearer $TOKEN" http://localhost:9090/version
# {"version":"v0.1.0","build_date":"2026-06-12T21:42:03Z","git_commit":"1fe1ea5","go_version":"go1.26.3","os":"linux","arch":"amd64"}
```

## Connecting via SSM

```bash
# Start port-forward
aws ssm start-session --target i-INSTANCE_ID \
  --document-name AWS-StartPortForwardingSession \
  --parameters '{"portNumber":["9090"],"localPortNumber":["9090"]}'

# In another terminal, access the agent
curl -H "Authorization: Bearer $TOKEN" http://localhost:9090/health
```

## Configuration

| Env Var | Default | Description |
|---------|---------|-------------|
| `AGENT_TOKEN_FILE` | | Path to file containing HMAC token |
| `AGENT_TOKEN` | | HMAC token (env var fallback) |
| `AGENT_PORT` | `9090` | Listen port |
| `AGENT_HOST` | `127.0.0.1` | Listen address |
| `AGENT_KUBECONFIG` | `/etc/kubernetes/admin.conf` | Kubeconfig path |

## Building

```bash
make build          # Cross-compile for linux/amd64
make build-local    # Build for current platform
make dev            # Build and run with dev token
make test           # Run tests
```

The build injects version, build date, and git commit via `-ldflags`.

## Security Model

- **No public listening** â€” binds to 127.0.0.1 only
- **SSM-only access** â€” all traffic tunneled through AWS SSM (IAM-authenticated)
- **HMAC auth** â€” every request requires a Bearer token
- **Command whitelist** â€” only approved binaries can execute
- **Rate limiting** â€” 100 req/min per IP
- **Body size limits** â€” 1MB max request body
- **Audit logging** â€” every request logged with method, path, status, duration, request ID
- **Input validation** â€” all inputs validated before execution
- **No shell injection** â€” `/exec/shell` passes commands to `bash -c` through the whitelist check
